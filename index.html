    Logic Explanation

    How the Array Works

    There's one array, lootItems, declared at the very top of script.js outside any function. That placement is key — it means the array survives between button clicks. JavaScript doesn't reset it every time someone clicks "Add Loot" the way a local variable would. Each item in the array is stored as a plain object with four properties: name, value (the multiplied final amount), baseValue (what the user typed), and rarity. Storing objects instead of raw numbers means I can render all the details later without keeping separate parallel arrays.

    Where Loops Are Used

    renderLoot() uses a traditional for loop that walks i from 0 up to lootItems.length - 1. Inside the loop, it does two things at once: adds each item's value to a running total variable, and builds a <li> element for that item and appends it to the list. splitLoot() uses its own separate for loop to re-total the array right before dividing — so the split is always based on a fresh calculation, not a number that might have drifted.

    Conditional Logic

    Validation runs entirely through if statements before anything is pushed to the array or divided. addLoot() checks three things in order: is the name empty, is the value a valid number, is the value negative — returning early with an error message the moment one fails. splitLoot() checks that party size is a number 1 or greater, and that lootItems.length is more than zero. renderLoot() checks lootItems.length === 0 to decide whether to show the empty-state message or render the live list.
